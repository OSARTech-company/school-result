# School Admin Handbook

## Purpose

This handbook explains what a school admin can do in the Student Score Management System and how the main school-admin menus are used.

The school admin is responsible for day-to-day school setup, academic coordination, communication, monitoring, and recovery tasks for one school tenant only.

## School Admin Responsibilities

The school admin can:

- Manage students, teachers, parents, subjects, timetable, and school settings.
- Publish, approve, reject, and correct results.
- Send school messages to students and teachers.
- Review requests, disputes, audits, and reports.
- Run backups, health checks, and recovery drills.
- Control school-specific access and operations.

The school admin cannot:

- Manage other schools.
- Access super-admin-only global controls.
- Bypass school access restrictions.
- Edit platform-wide settings for all schools.

## Main Menu Guide

### 1. Dashboard

The dashboard is the school admin home page.

Use it to:

- View school status and alerts.
- Open the main school workflows.
- Track pending actions and quick summaries.

### 2. Student Menu

Use this page to search, filter, and review student records.

- View students by class and term.
- Open student details.
- Access result and correction actions where allowed.

#### Add Students

You can:

- Add students manually.
- Import students by CSV.
- Review skipped rows and validation messages.

#### Reset Requests

Use this page to review student password reset requests.

#### Subject Requests

Use this page to review student subject add/drop requests.

#### View Parents

Use this page to review linked parent records for students.

#### Parent Directory

Use this page to browse parent contact information and links.

## Recommended School Admin Workflow

For a new school setup, use this order:

1. Update `School Settings`
2. Configure `Class Subjects`
3. Add `Teachers`
4. Use `Assign Teacher`
5. Add `Students`
6. Set up `Timetable`
7. Create `Term Programs`
8. Test `Messages`
9. Review `Analytics` and `Data Integrity`
10. Take a `Backup`

## Important Notes

- School admin actions only apply to the current school session.
- Some actions are blocked when school operations are restricted.
- Results, timetable, and message workflows should be checked carefully before saving.
- Keep backups before major bulk changes.
